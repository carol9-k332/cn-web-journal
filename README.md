# cn-web-journal

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要作为静态 HTML 页面的集合与发布点，不针对特定域名或具体网站。通过将页面以文件形式存放于此，便于长期保存、版本管理及直接通过 GitHub Pages 等静态托管服务访问。页面内容涵盖各类信息记录、存档或实验性展示，力求保持原始结构与样式。

## 目录说明

```
cn-web-journal/
├── pages/        # 存放所有独立 HTML 页面文件
├── assets/       # 页面引用的静态资源（图片、样式、脚本等）
├── index.html    # 可选的主入口页面
└── README.md     # 本说明文件
```

- `pages/`：每个 HTML 文件为一个独立页面，文件名通常反映内容主题或归档日期。
- `assets/`：按需存放页面所需的公共或私有资源，建议按目录组织。
- 仓库根目录下的 `index.html` 可作为导航或索引页，非必需。

## 页面归档说明

归档的 HTML 页面遵循以下原则：

- 每个页面保持完整独立，不依赖外部动态服务。
- 内容以静态 HTML 为主，可包含少量 CSS 和 JavaScript。
- 页面之间无强制链接关系，各自独立运行。
- 文件命名建议使用英文小写、数字及连字符，避免特殊字符。

如需添加新页面，直接将 HTML 文件放入 `pages/` 目录，并更新资源引用路径即可。

## 维护说明
